## 1. 文本分类到底在做什么

文本分类的任务是：

> 给定一段文本，判断它属于哪个类别。

例子：

| 文本 | 类别 |
| --- | --- |
| “央行宣布新的利率政策” | 财经 |
| “球队在决赛中逆转取胜” | 体育 |
| “新款手机发布，搭载最新芯片” | 科技 |
| “这家餐厅服务很好，但价格偏高” | 评论情感：中性或正面 |

如果用数学语言表达：

```text
输入：文本 x
输出：类别 y
模型：f(x) = y
```

更进一步，模型通常不是直接给一个类别，而是给每个类别一个概率：

```text
P(财经 | x) = 0.80
P(体育 | x) = 0.05
P(科技 | x) = 0.10
P(娱乐 | x) = 0.05
```

最后选概率最大的类别：

```text
预测类别 = argmax_y P(y | x)
```

### 1.1 一流课堂会先问什么

优秀教授通常不会先问“用什么模型”，而会先问：

- 类别是什么？
- 类别之间是否互斥？
- 一篇文章能不能属于多个类别？
- 文本长度差异大不大？
- 类别是否平衡？
- 错分不同类别的代价是否一样？
- 你希望模型可解释，还是只追求准确率？

优秀学生会意识到：

> 模型选择不是第一步，问题定义才是第一步。

### 1.2 单标签、多标签与层级分类

单标签分类：

```text
一篇文章只属于一个类别。
```

例如新闻分类：财经、体育、科技、娱乐。

多标签分类：

```text
一篇文章可以同时属于多个类别。
```

例如一篇文章可以同时是“科技”“商业”“人工智能”。

层级分类：

```text
类别有上下级关系。
```

例如：

```text
体育
├── 篮球
├── 足球
└── 网球
```

不同任务决定不同数学形式：

- 单标签：常用 Softmax。
- 多标签：常用多个 Sigmoid。
- 层级分类：要考虑父子类别约束。

## 2. 第一步：把文本变成可计算的对象

计算机不能直接理解“我喜欢这部电影”。  
它只能处理数字。

所以文本分类第一步是：

> 把文本转成数字向量。

### 2.1 分词

英文天然有空格：

```text
I love machine learning
```

可以分成：

```text
["I", "love", "machine", "learning"]
```

中文没有天然空格：

```text
我喜欢机器学习
```

可能分成：

```text
["我", "喜欢", "机器学习"]
```

也可以按字：

```text
["我", "喜", "欢", "机", "器", "学", "习"]
```

分词会影响后续所有步骤。

教授式问题：

> 一个词到底是什么？中文里“机器学习”应该是一个词，还是四个字？

这不是细节，而是表示方式的一部分。

### 2.2 词表

把训练集中出现过的词收集起来，形成词表：

```text
["央行", "利率", "球队", "比赛", "手机", "芯片"]
```

给每个词一个编号：

```text
央行 -> 0
利率 -> 1
球队 -> 2
比赛 -> 3
手机 -> 4
芯片 -> 5
```

于是文本：

```text
央行 利率
```

可以变成：

```text
[0, 1]
```

这只是编号，还不是模型真正需要的向量。

### 2.3 One-hot 表示

如果词表大小是 6，那么“央行”可以表示为：

```text
[1, 0, 0, 0, 0, 0]
```

“芯片”可以表示为：

```text
[0, 0, 0, 0, 0, 1]
```

One-hot 的问题：

1. 向量很长。
2. 大部分位置是 0。
3. 看不出词和词之间的语义关系。

在 One-hot 里，“手机”和“芯片”的距离，和“手机”与“球队”的距离一样远。  
这显然不符合人类理解。

## 3. 词袋模型：最朴素但很重要的表示

词袋模型的思想：

> 不管词的顺序，只统计每个词出现了多少次。

例如词表：

```text
["央行", "利率", "球队", "比赛", "手机", "芯片"]
```

文章：

```text
央行 调整 利率 利率
```

如果只看词表中的词，向量可能是：

```text
[1, 2, 0, 0, 0, 0]
```

含义：

- “央行”出现 1 次。
- “利率”出现 2 次。
- 其他词出现 0 次。

### 3.1 为什么词袋模型有效

虽然它忽略顺序，但对很多分类任务已经很强。

原因是：

> 类别常常由关键词决定。

财经文章里常出现：

```text
央行、利率、股票、市场、基金
```

体育文章里常出现：

```text
比赛、球队、冠军、教练、比分
```

只要关键词足够明显，词袋模型就能工作。

### 3.2 词袋模型的缺点

词袋模型看不到顺序。

下面两句话词袋非常接近：

```text
我喜欢这部电影
我不喜欢这部电影
```

如果“不”没有被模型正确重视，就可能分错。

词袋模型也看不到复杂语义：

```text
这部电影并不是不好看
```

它包含“不”“不好看”，但整体可能是正面评价。

优秀学生会总结：

> 词袋模型适合关键词驱动的分类，不适合强依赖语序和上下文的理解。

## 4. TF-IDF：让“有区分度的词”更重要

词频只看一个词在当前文章出现多少次。  
但有些词出现很多，却没有分类价值。

例如：

```text
的、了、是、我们、这个
```

这些词在各种文章里都常见，不能很好区分类别。

TF-IDF 的思想：

> 一个词在当前文章中出现多，说明重要；但如果它在所有文章中都常见，就降低它的重要性。

### 4.1 TF：词频

```text
TF = 某词在当前文章中出现次数
```

也可以除以文章总词数，避免长文章天然词频更高。

### 4.2 IDF：逆文档频率

```text
IDF(word) = log(总文章数 / 包含该词的文章数)
```

如果一个词出现在几乎所有文章里：

```text
包含该词的文章数很大
IDF 很小
```

如果一个词只出现在少数文章里：

```text
包含该词的文章数很小
IDF 很大
```

### 4.3 TF-IDF

```text
TF-IDF(word, document) = TF(word, document) * IDF(word)
```

直觉：

- 当前文章里常出现，加分。
- 全局到处都出现，减分。

### 4.4 教授式理解

教授可能会问：

> “重要”到底是什么意思？

TF-IDF 的回答是：

> 一个词如果能把当前文章和其他文章区分开，它就更重要。

这是一种非常关键的机器学习思想：

> 特征的价值来自区分能力。

## 5. 朴素贝叶斯：用概率做分类

朴素贝叶斯是文本分类中的经典方法，尤其适合垃圾邮件、新闻主题分类等任务。

它想计算：

```text
P(类别 | 文本)
```

例如：

```text
P(体育 | "球队 冠军 比赛") 
```

### 5.1 贝叶斯公式

贝叶斯公式：

```text
P(y | x) = P(x | y) * P(y) / P(x)
```

用于分类时，`P(x)` 对所有类别一样，所以只比较：

```text
P(y | x) ∝ P(x | y) * P(y)
```

含义：

- `P(y)`：这个类别本身出现的概率。
- `P(x | y)`：在这个类别下看到这段文本的概率。

### 5.2 朴素假设

文本由多个词组成：

```text
x = [w1, w2, w3, ...]
```

朴素贝叶斯假设：

> 在给定类别后，词与词之间相互独立。

于是：

```text
P(x | y) = P(w1 | y) * P(w2 | y) * P(w3 | y) * ...
```

这个假设显然不完全真实。  
例如“机器”和“学习”并不独立。

但它仍然常常有效，因为分类不需要完美理解语言，只需要抓住足够强的统计信号。

### 5.3 一个直观例子

如果训练集中：

```text
体育类文章里，“球队”出现很多
财经类文章里，“球队”很少出现
```

那么：

```text
P("球队" | 体育) > P("球队" | 财经)
```

当新文章包含“球队”“比赛”“冠军”时，体育类别的概率会被不断放大。

### 5.4 为什么要平滑

如果某个词在某个类别中从未出现：

```text
P(word | class) = 0
```

那么整个乘积都会变成 0。

这太极端。  
训练集中没出现，不代表未来绝不可能出现。

所以常用拉普拉斯平滑：

```text
P(word | class) = (count(word, class) + 1) / (total_words_in_class + vocab_size)
```

直觉：

> 给每个词一点点基础概率，避免因为数据有限而断言“不可能”。

### 5.5 朴素贝叶斯的优缺点

优点：

- 简单。
- 快。
- 数据不多时也能工作。
- 可解释性强。

缺点：

- 独立性假设过强。
- 不擅长复杂语义。
- 对词序、上下文理解弱。

教授式总结：

> 朴素贝叶斯不是因为假设真实才有用，而是因为它用很低成本抓住了类别和词之间的强相关性。

## 6. 逻辑回归：用一条边界分开文本

逻辑回归是非常重要的线性分类模型。

它的输入是文本向量：

```text
x = [x1, x2, ..., xn]
```

模型有一组权重：

```text
w = [w1, w2, ..., wn]
```

计算分数：

```text
score = w · x + b
```

也就是：

```text
score = w1*x1 + w2*x2 + ... + wn*xn + b
```

### 6.1 权重是什么意思

如果 `x_i` 表示某个词的 TF-IDF 值，`w_i` 表示模型给这个词的倾向。

例如判断“体育”：

- “球队”的权重可能很大。
- “冠军”的权重可能很大。
- “利率”的权重可能为负。

模型在做的事情是：

> 把所有词的证据加起来。

### 6.2 Sigmoid：把分数变成概率

二分类时，用 Sigmoid：

```text
sigmoid(z) = 1 / (1 + e^(-z))
```

它把任意实数变成 `0..1` 之间的数。

```text
z 很大 -> 概率接近 1
z 很小 -> 概率接近 0
z = 0 -> 概率 = 0.5
```

### 6.3 损失函数

模型需要知道自己错得多严重。

二分类常用交叉熵损失：

```text
Loss = -[y log(p) + (1-y) log(1-p)]
```

如果真实类别是 1，但模型只预测 `p = 0.01`，损失会很大。  
如果真实类别是 1，模型预测 `p = 0.99`，损失很小。

### 6.4 优秀学生如何理解逻辑回归

不要把逻辑回归理解成公式堆叠。  
它可以被理解为：

> 每个词给不同类别投票，模型学习每个词的投票权重。

当训练完成后，权重大且为正的词，就是强正证据。  
权重大且为负的词，就是强反证据。

这也是为什么逻辑回归常被用作可解释基线模型。

## 7. 支持向量机：寻找最稳的分界线

支持向量机常用于传统文本分类。

它的核心思想：

> 不只是把样本分开，还要让分界线离两边最近的样本尽量远。

这个距离叫间隔。

### 7.1 为什么间隔重要

如果分界线紧贴某些样本，那么新样本稍微有点变化就可能被分错。

更大的间隔意味着：

```text
模型更稳
对噪声更不敏感
泛化可能更好
```

教授式解释：

> 不是刚好答对训练题就好，而是要答得有余量。

### 7.2 文本分类中为什么 SVM 常有效

文本向量通常维度很高。

例如词表有 50000 个词，每篇文章是 50000 维稀疏向量。

在线性高维空间中，很多类别确实可以被相对简单的线性边界分开。  
所以线性 SVM 在传统文本分类中非常强。

## 8. 词向量：让词拥有语义位置

词袋和 TF-IDF 的问题是：看不出语义相似。

词向量的目标：

> 把词映射到一个连续向量空间，让语义相近的词距离更近。

例如：

```text
vector("手机") 和 vector("芯片") 比较近
vector("手机") 和 vector("球队") 比较远
```

### 8.1 分布式假设

词向量背后的重要思想：

> 一个词的意义由它经常出现的上下文决定。

如果两个词经常出现在相似上下文中，它们语义可能相近。

例如：

```text
我买了一部 手机
我买了一台 电脑
```

“手机”和“电脑”出现在类似上下文里，所以向量会更接近。

### 8.2 从词向量到文章向量

有了词向量后，文章可以表示成多个词向量的组合。

最简单做法：

```text
文章向量 = 所有词向量的平均值
```

更复杂做法：

- RNN 按顺序读词。
- CNN 抓局部短语。
- Transformer 根据上下文动态计算词表示。

### 8.3 静态词向量的问题

同一个词在不同语境中可能意思不同。

例如：

```text
苹果 发布了新手机
我吃了一个 苹果
```

静态词向量会给两个“苹果”同一个向量。  
上下文模型会根据句子给它们不同表示。

## 9. 深度学习文本分类

深度学习文本分类一般流程：

```text
文本 -> Token -> 向量 -> 神经网络 -> 类别概率
```

### 9.1 嵌入层

嵌入层把 token 编号变成向量。

```text
token_id -> embedding vector
```

例如：

```text
"手机" -> [0.12, -0.03, 0.88, ...]
```

这些向量可以随机初始化，然后通过训练学习。  
也可以使用预训练模型提供的向量。

### 9.2 文本 CNN

文本 CNN 用卷积窗口抓局部短语。

例如窗口大小为 3，可以看：

```text
["非常", "值得", "推荐"]
["不", "值得", "购买"]
```

它擅长捕捉局部模式。

### 9.3 RNN 和 LSTM

RNN 按顺序读文本：

```text
第 1 个词 -> 第 2 个词 -> 第 3 个词 -> ...
```

它适合处理顺序，但长文本中早期信息容易衰减。  
LSTM 和 GRU 通过门控机制缓解这个问题。

### 9.4 深度学习的优势

相比 TF-IDF + 线性模型，深度模型可以学习：

- 词序。
- 短语模式。
- 上下文语义。
- 非线性特征组合。

例如：

```text
不是不好看
```

深度模型更有机会理解这种组合含义。

## 10. Transformer 与注意力机制

Transformer 是现代文本模型的核心结构。

它的关键是注意力机制：

> 一个词在理解自己时，可以主动关注句子中其他相关词。

### 10.1 为什么需要注意力

看句子：

```text
这部电影虽然节奏慢，但结尾非常震撼。
```

判断情感时，“震撼”很重要，“虽然”提示前后有转折。

模型不能只看词频，也不能只按顺序机械读取。  
它需要知道哪些词对当前判断更重要。

### 10.2 Query、Key、Value 的直觉

注意力里有三个向量：

- Query：我现在想找什么信息。
- Key：我能提供什么索引。
- Value：我真正携带的信息。

计算过程：

1. 当前词拿 Query 去和其他词的 Key 比较。
2. 相似度越高，注意力权重越大。
3. 用权重加权求和 Value。

公式简化：

```text
Attention(Q, K, V) = softmax(QK^T / sqrt(d)) V
```

直觉：

> 先算“该看谁”，再把看到的信息汇总回来。

### 10.3 自注意力

自注意力表示：

> 同一句话里的词互相看。

每个词都可以根据整句话更新自己的表示。

例如：

```text
苹果 发布了 新手机
```

“苹果”会关注“发布”“手机”，从而更像公司名。

```text
我 吃了 一个 苹果
```

“苹果”会关注“吃”，从而更像水果。

### 10.4 Transformer 为什么强

原因包括：

- 能捕捉长距离依赖。
- 并行计算效率高。
- 多层结构能逐步组合语义。
- 预训练能从海量文本中学习语言规律。

教授式提醒：

> Transformer 并不是魔法，它只是用大量数据、强表达能力和可训练结构，把上下文相关性学得非常好。

## 11. 评估指标：模型到底好不好

准确率不是唯一指标。

### 11.1 混淆矩阵

二分类中：

| 预测/真实 | 正类 | 负类 |
| --- | --- | --- |
| 预测正类 | TP | FP |
| 预测负类 | FN | TN |

含义：

- TP：真正例。
- FP：假正例。
- FN：假负例。
- TN：真负例。

### 11.2 Precision、Recall、F1

Precision：

```text
预测为正的样本中，有多少是真的正
```

Recall：

```text
真实为正的样本中，有多少被找出来
```

F1：

```text
Precision 和 Recall 的调和平均
```

如果做垃圾邮件识别：

- Precision 低：正常邮件容易被误判为垃圾。
- Recall 低：很多垃圾邮件漏掉。

不同业务对错误的容忍不同。

### 11.3 类别不平衡

如果 99% 的文章都是“非垃圾”，模型全部预测“非垃圾”，准确率也有 99%。  
但它没有任何实际价值。

所以类别不平衡时，要看：

- 每类 Precision。
- 每类 Recall。
- Macro F1。
- Weighted F1。
- ROC-AUC 或 PR-AUC。

优秀学生会问：

> 这个指标是否真的反映了任务目标？

## 12. 教授式思考流程

面对一个文本分类任务，按这个流程思考。

### 12.1 先定义任务

```text
类别是什么？
单标签还是多标签？
是否有层级？
错分代价是否相同？
```

### 12.2 再观察数据

```text
每类有多少样本？
文本平均长度是多少？
是否有大量噪声？
是否有重复样本？
是否有时间变化？
```

### 12.3 再建立基线

不要一开始就上复杂模型。

推荐顺序：

1. 规则关键词。
2. TF-IDF + 朴素贝叶斯。
3. TF-IDF + 逻辑回归或 SVM。
4. 词向量 + 简单神经网络。
5. Transformer 微调。

这样可以知道复杂模型到底带来了多少提升。

### 12.4 最后分析错误

抽样看错分文本，问：

- 是标签本身有歧义吗？
- 是文本太短吗？
- 是需要上下文吗？
- 是类别定义重叠吗？
- 是训练数据太少吗？
- 是模型没有见过新词吗？

优秀学生不是只看总体分数，而是通过错误样本理解模型边界。

## 13. 初学者训练路线

### 第一阶段：理解传统模型

学习目标：

- 分词。
- 词表。
- 词袋。
- TF-IDF。
- 朴素贝叶斯。
- 逻辑回归。

练习：

```text
用 TF-IDF + 逻辑回归做新闻分类。
观察每个类别权重最高的词。
```

### 第二阶段：理解向量语义

学习目标：

- 词向量。
- 文章向量。
- 相似度。
- 语义空间。

练习：

```text
找出和“手机”“股票”“球队”最相近的词。
观察模型是否学到了直觉上的语义关系。
```

### 第三阶段：理解深度模型

学习目标：

- 嵌入层。
- CNN/RNN 文本分类。
- Transformer。
- 注意力。

练习：

```text
对比 TF-IDF 模型和 Transformer 模型在短文本、长文本、反讽文本上的错误。
```

### 第四阶段：形成工程判断

学习目标：

- 数据清洗。
- 类别不平衡处理。
- 指标选择。
- 错误分析。
- 模型可解释性。

最终能力：

> 看到一个文本分类任务，能判断它是关键词驱动、语义驱动、上下文驱动，还是标签定义本身存在问题。

---

## 一句话总结

文本分类的核心路径是：

```text
文本 -> 数字表示 -> 统计或神经模型 -> 类别概率 -> 评估与错误分析
```

从词袋到 Transformer，复杂度不断增加，但根本问题没有变：

> 如何把文本中对类别有用的信息提取出来，并用数学模型稳定地做判断。
