## 0. 如何使用本文：图论学习路线

图论不应该只按“背算法模板”的方式学习。

更好的路线是：

```text
先理解关系如何变成图；
再掌握基本定义；
再学习图的结构定理；
再学习算法如何利用这些结构；
最后训练证明和建模能力。
```

很多初学者会这样学图论：

```text
今天背 BFS；
明天背 DFS；
后天背 Dijkstra；
遇到新题时问“这题套哪个模板”。
```

这种方法能解决一部分标准题，但遇到陌生问题会困难。

更接近 MIT、CMU 理论课程的训练方式是：

```text
先问图的对象是什么；
边表示什么关系；
目标是可达、最短、最小连接、最大配对、冲突最少，还是流量最大；
然后根据图结构选择算法；
最后用不变量、割性质、层次性质或归纳证明正确性。
```

### 0.1 五遍学习法

| 阶段 | 目标 | 应读章节 | 能力标准 |
| --- | --- | --- | --- |
| 第一遍：认识图 | 理解点、边、度、路径、连通 | 1 到 6 | 能把现实关系画成图并说清定义 |
| 第二遍：会遍历 | 掌握 BFS、DFS、树、二分图、DAG | 7 到 11 | 能处理可达性、层次、环检测、拓扑顺序 |
| 第三遍：会优化 | 掌握最短路、最小生成树、并查集、SCC | 12 到 15 | 能解决距离、连接成本、动态连通、有向互达问题 |
| 第四遍：会结构化证明 | 掌握欧拉、哈密顿、平面图、着色、匹配、流 | 16 到 22 | 能写出反例、必要条件、充分条件和正确性证明 |
| 第五遍：会建模迁移 | 从陌生题抽象图模型并选择算法 | 23 到 26 | 能独立分析新题，而不是依赖背题 |

### 0.2 图论题的教授式五问

拿到一个图论问题，先问：

```text
1. 顶点是什么？
2. 边是什么？
3. 边有方向吗？
4. 边有权重或容量吗？
5. 我要找的是可达、最短、最小连接、最大匹配、最优流，还是证明某种结构存在？
```

例如“课程先修关系”：

```text
顶点：课程；
边：A -> B 表示学 B 前必须先学 A；
方向：有方向；
目标：判断是否有环，若无环则给出学习顺序；
模型：DAG + 拓扑排序。
```

例如“社交网络中两人最少经过几层朋友关系能认识”：

```text
顶点：人；
边：朋友关系；
方向：通常无向；
权重：无权；
目标：最少边数；
模型：无权图 BFS。
```

### 0.3 优秀学生的图论笔记格式

每学一个定理或算法，都写成：

```text
定义：
直觉：
小例子：
反例：
算法：
不变量：
正确性：
复杂度：
适用边界：
```

例如 BFS：

```text
定义：按距离源点的层数扩展。
直觉：先访问近的，再访问远的。
不变量：队列中节点距离不递减。
正确性：第一次访问某点时已经得到无权最短距离。
复杂度：邻接表 O(V + E)。
边界：只适合无权图或所有边权相同的图。
```

这比只背模板更稳定。

## 1. 图论到底研究什么

图论研究对象之间的关系。

一个图由：

```text
顶点 vertex：对象；
边 edge：对象之间的关系；
```

组成。

图论的强大之处在于：

```text
很多看似不同的问题，本质上都是“对象和关系”的问题。
```

例如：

| 现实问题 | 顶点 | 边 | 图论问题 |
| --- | --- | --- | --- |
| 地图导航 | 城市或路口 | 道路 | 最短路径 |
| 社交网络 | 用户 | 好友关系 | 连通性、中心性 |
| 课程安排 | 课程 | 先修关系 | 拓扑排序 |
| 任务调度 | 任务 | 依赖关系 | DAG 动态规划 |
| 网络通信 | 路由器 | 链路 | 最短路、最小割 |
| 项目分组 | 人 | 冲突关系 | 图着色 |
| 二分匹配 | 学生和项目 | 可选择关系 | 最大匹配 |
| 电路和网页 | 元件或页面 | 连接或链接 | 有向图分析 |

### 1.1 图论和算法的关系

离散数学中的图论偏向：

```text
定义；
性质；
存在性；
计数；
证明；
反例；
```

算法课程中的图算法偏向：

```text
如何表示图；
如何遍历图；
如何计算最优路径；
如何维护连通性；
如何在约束下最大化或最小化；
如何证明算法正确和分析复杂度；
```

真正的图论能力要把两者结合。

例如最小生成树不是只会 Kruskal 模板，而要理解：

```text
为什么任意割上的最轻边是安全的；
为什么加入这条边不会破坏最优性；
为什么并查集能高效维护“是否已经连通”；
```

### 1.2 图论为什么适合训练抽象能力

图论训练的是把具体问题抽象成结构的能力。

优秀学生通常会先做抽象，而不是先写代码。

例如题目说：

```text
有 n 个房间，一些房间之间有门，问能否从 0 号房间到所有房间。
```

抽象：

```text
房间是顶点；
门是边；
问题是从源点出发的可达性；
算法是 BFS 或 DFS；
```

题目说：

```text
给出若干不等式 x < y，问是否能给变量排序。
```

抽象：

```text
变量是顶点；
x < y 是有向边 x -> y；
是否存在排序等价于图是否无环；
若无环，用拓扑排序。
```

这就是图论的本质：把复杂叙述翻译成点和边。

## 2. 一句话理解图论

图论可以理解为：

```text
图论 = 用点表示对象 + 用边表示关系 + 研究关系形成的结构和可计算问题
```

图算法可以理解为：

```text
图算法 = 在点和边组成的关系网络中，高效回答可达、距离、连接、顺序、匹配、流量、冲突等问题
```

### 2.1 图论中的核心问题

常见问题可以归成几类：

| 问题类型 | 典型算法或定理 |
| --- | --- |
| 能不能到达 | BFS、DFS、连通分量 |
| 最少走几步 | BFS |
| 最小代价路径 | Dijkstra、Bellman-Ford、Floyd |
| 是否有环 | DFS、拓扑排序 |
| 依赖能否安排 | DAG、拓扑排序 |
| 如何最低成本连通 | 最小生成树 |
| 如何判断两类冲突 | 二分图染色 |
| 如何最大配对 | 二分图匹配 |
| 如何最大运输 | 最大流 |
| 能否不重复走完所有边 | 欧拉路 |
| 能否不重复访问所有点 | 哈密顿问题 |
| 地图能否无交叉画出 | 平面图 |
| 最少用多少颜色避免冲突 | 图着色 |

### 2.2 图论的两个基本视角

第一种视角是局部视角：

```text
某个点有哪些邻居？
某条边连接谁？
某个点度数是多少？
```

第二种视角是全局视角：

```text
整张图是否连通？
是否存在环？
是否能分成两边？
是否有拓扑顺序？
是否能用少量颜色染色？
是否存在覆盖所有点或边的路线？
```

很多图论证明就是从局部性质推出全局结构。

例如握手定理：

```text
每条边贡献两个端点度数；
所以所有点度数之和等于 2|E|；
于是奇数度顶点个数必须是偶数。
```

这是典型的离散数学思维：同一个量用两种方式计数。

## 3. 图的基本定义

图通常写作：

```text
G = (V, E)
```

其中：

```text
V 是顶点集合；
E 是边集合；
```

如果是无向图：

```text
边 {u, v} 表示 u 和 v 之间没有方向。
```

如果是有向图：

```text
边 (u, v) 表示从 u 指向 v。
```

### 3.1 无向图

无向图适合表达对称关系：

```text
朋友关系；
道路双向通行；
两个变量冲突；
两个城市之间有航线且不考虑方向；
```

示例：

```text
V = {A, B, C}
E = {{A, B}, {B, C}}
```

图形：

```text
A -- B -- C
```

### 3.2 有向图

有向图适合表达非对称关系：

```text
课程先修；
网页链接；
函数调用；
任务依赖；
单向道路；
状态转移；
```

示例：

```text
A -> B -> C
```

表示可以从 A 到 B，从 B 到 C，但不一定能从 C 回到 A。

### 3.3 加权图

如果边带权重：

```text
w(u, v) = 边的代价、距离、时间、费用、容量或风险。
```

示例：

```text
A --5-- B --2-- C
```

最短路径问题通常需要加权图。

注意：

```text
无权图可以看作每条边权重都是 1；
BFS 适合无权最短路；
Dijkstra 适合非负权最短路；
Bellman-Ford 可以处理负权边，但不能处理可达负环下的有限最短路；
```

### 3.4 简单图、多重图与自环

简单图：

```text
没有自环；
任意两个顶点之间最多一条边；
```

自环：

```text
边从一个点连回自己，例如 (u, u)。
```

多重边：

```text
同一对顶点之间有多条边。
```

初学阶段多数定理默认简单图。

遇到题目时要先确认：

```text
是否允许自环？
是否允许重边？
是否有方向？
是否有权重？
```

这些边界会影响算法和定理是否成立。

### 3.5 完全图、空图、子图、补图

完全图 `K_n`：

```text
n 个顶点两两相连。
```

边数：

```text
|E(K_n)| = n(n - 1) / 2
```

空图：

```text
有顶点但没有边。
```

子图：

```text
从原图取一部分顶点和边形成的图。
```

诱导子图：

```text
取一部分顶点，并保留原图中这些顶点之间所有边。
```

补图：

```text
在同一个顶点集合上，原图没有的边在补图中有，原图有的边在补图中没有。
```

这些概念常用于证明和构造反例。

## 4. 图的表示：邻接表、邻接矩阵与边表

算法实现前，必须决定图如何存储。

常用三种表示：

```text
邻接表；
邻接矩阵；
边表；
```

选择取决于：

```text
顶点数 V；
边数 E；
需要频繁遍历邻居还是查询两点是否相连；
图是稠密还是稀疏；
```

### 4.1 邻接表

邻接表为每个顶点保存邻居列表。

无向图：

```text
0: 1, 2
1: 0, 3
2: 0
3: 1
```

Python 风格伪代码：

```python
graph = [[] for _ in range(n)]

for u, v in edges:
    graph[u].append(v)
    graph[v].append(u)
```

有向图：

```python
graph = [[] for _ in range(n)]

for u, v in edges:
    graph[u].append(v)
```

复杂度：

```text
空间：O(V + E)
遍历所有边：O(V + E)
查询 u 和 v 是否相邻：最坏 O(deg(u))
```

适合大多数稀疏图。

### 4.2 邻接矩阵

邻接矩阵用 `V x V` 矩阵表示边。

```text
matrix[u][v] = 1 表示 u 到 v 有边。
matrix[u][v] = weight 表示边权。
```

复杂度：

```text
空间：O(V^2)
查询 u 和 v 是否相邻：O(1)
遍历 u 的所有邻居：O(V)
```

适合：

```text
顶点数较小；
图很稠密；
需要频繁查询两点是否相连；
Floyd-Warshall 这类矩阵式算法；
```

不适合：

```text
V 很大但 E 很小的稀疏图。
```

### 4.3 边表

边表直接保存所有边：

```text
[(u1, v1, w1), (u2, v2, w2), ...]
```

适合：

```text
Kruskal 最小生成树；
Bellman-Ford 松弛所有边；
读入和排序边；
```

复杂度：

```text
空间：O(E)
遍历所有边：O(E)
查某个点的邻居：不方便；
```

### 4.4 表示选择清单

| 需求 | 推荐表示 |
| --- | --- |
| 遍历邻居 | 邻接表 |
| 稀疏图 | 邻接表 |
| 快速判断两点是否相连 | 邻接矩阵 |
| 顶点数小且稠密 | 邻接矩阵 |
| 按权重排序所有边 | 边表 |
| Bellman-Ford | 边表或邻接表 |
| Kruskal | 边表 + 并查集 |

优秀学生会先估算：

```text
V 有多大？
E 有多大？
是否需要 O(1) 查边？
是否需要遍历所有邻居？
```

这一步决定后面算法能不能过。

## 5. 度、握手定理与计数思维

顶点的度表示它连接了多少条边。

在无向图中：

```text
deg(v) = 与 v 相 incident 的边数。
```

如果有自环，通常一个自环对度贡献 2。

### 5.1 握手定理

无向图中：

```text
所有顶点度数之和 = 2 × 边数
```

写作：

```text
sum(deg(v) for v in V) = 2|E|
```

为什么？

```text
每条边有两个端点；
统计所有顶点的度时，每条边被数了两次；
```

这是图论中最经典的“双重计数”。

### 5.2 奇度顶点个数为偶数

因为所有度数之和是偶数。

偶数度顶点贡献偶数。

如果奇数度顶点个数是奇数，那么总和会是奇数，矛盾。

所以：

```text
任意无向图中，奇数度顶点的个数一定是偶数。
```

这条结论常用于欧拉路判断。

### 5.3 有向图中的入度和出度

有向图中：

```text
indeg(v) = 指向 v 的边数；
outdeg(v) = 从 v 指出的边数；
```

总入度等于总出度：

```text
sum(indeg(v)) = sum(outdeg(v)) = |E|
```

因为每条有向边都有一个起点和一个终点。

### 5.4 度数序列

图的度数序列是所有顶点度数排成的序列。

例如：

```text
路径 P4：1, 2, 2, 1
环 C4：2, 2, 2, 2
星形图 K1,4：4, 1, 1, 1, 1
```

度数序列能提供必要信息，但不能完全决定图。

两个非同构图可能有相同度数序列。

这就是反例意识：

```text
相同局部统计不一定推出相同全局结构。
```

## 6. 路、迹、回路、环与连通性

图论中很多概念都和“走”有关。

### 6.1 walk、trail、path、cycle

中文教材翻译不完全统一，学习时要看定义。

常见区分：

```text
walk：可以重复点和边的走法；
trail：不重复边的走法；
path：不重复顶点的走法；
cycle：起点终点相同，除起终点外不重复顶点的闭合 path；
```

初学时可以记：

```text
path 更严格，不重复点；
trail 只要求不重复边；
walk 最宽松；
cycle 是闭合路径；
```

### 6.2 连通性

无向图中，如果任意两个顶点之间都有路径，则图连通。

连通分量：

```text
图中最大的连通子图。
```

例如：

```text
0 -- 1 -- 2     3 -- 4
```

有两个连通分量：

```text
{0, 1, 2}
{3, 4}
```

### 6.3 有向图的可达性

有向图中，从 u 到 v 有路径，不代表从 v 到 u 有路径。

强连通：

```text
任意两个顶点 u, v，都有 u -> v 和 v -> u 的有向路径。
```

强连通分量：

```text
有向图中互相可达的最大顶点集合。
```

### 6.4 割点、桥与脆弱连接

桥：

```text
删除这条边后，连通分量数量增加。
```

割点：

```text
删除这个点及相关边后，连通分量数量增加。
```

它们表达网络脆弱性。

例如：

```text
如果一条光缆断了，网络被分成两块，这条边就是桥。
```

DFS 时间戳可以在线性时间找桥和割点。

本文不展开完整 Tarjan 细节，但要记住：

```text
DFS 不只是遍历，也能发现图的结构。
```

## 7. BFS：按层扩展与无权最短路

BFS 是 breadth-first search，广度优先搜索。

核心直觉：

```text
从源点开始，先访问距离 1 的点，再访问距离 2 的点，再访问距离 3 的点。
```

所以 BFS 天然适合无权最短路。

### 7.1 BFS 模板

```python
from collections import deque

def bfs(graph, s):
    n = len(graph)
    dist = [-1] * n
    parent = [-1] * n
    q = deque()

    dist[s] = 0
    q.append(s)

    while q:
        u = q.popleft()
        for v in graph[u]:
            if dist[v] == -1:
                dist[v] = dist[u] + 1
                parent[v] = u
                q.append(v)

    return dist, parent
```

### 7.2 BFS 的不变量

BFS 的关键不变量：

```text
队列中的顶点按距离源点的层次非递减排列。
```

当一个顶点第一次被发现时：

```text
它的距离就是从源点到它的最短边数。
```

为什么？

```text
BFS 先处理所有距离 d 的点，才会处理距离 d+1 的点；
如果存在更短路径，它应该更早被发现；
第一次发现时没有更短路径被漏掉；
```

### 7.3 BFS 能解决的问题

```text
无权图最短路径；
网格最短步数；
社交网络几度好友；
迷宫最短出口；
判断连通分量；
二分图染色；
状态空间最少操作次数；
```

### 7.4 BFS 的边界

BFS 不适合一般加权最短路。

反例：

```text
A -> B 权重 100
A -> C 权重 1
C -> B 权重 1
```

BFS 会认为 A 到 B 一条边最短，但按权重看 A-C-B 更短。

所以：

```text
无权或等权：BFS；
非负权：Dijkstra；
存在负权：Bellman-Ford 或其他方法；
```

### 7.5 网格 BFS

网格常被看作隐式图。

```text
每个格子是顶点；
上下左右可走关系是边；
障碍格子不作为可达顶点；
```

```python
from collections import deque

def shortest_path_grid(grid, start):
    m, n = len(grid), len(grid[0])
    dist = [[-1] * n for _ in range(m)]
    q = deque([start])
    dist[start[0]][start[1]] = 0
    dirs = [(1, 0), (-1, 0), (0, 1), (0, -1)]

    while q:
        x, y = q.popleft()
        for dx, dy in dirs:
            nx, ny = x + dx, y + dy
            if 0 <= nx < m and 0 <= ny < n:
                if grid[nx][ny] != '#' and dist[nx][ny] == -1:
                    dist[nx][ny] = dist[x][y] + 1
                    q.append((nx, ny))

    return dist
```

教授式思考：

```text
不要先背“网格 BFS 模板”；
先说清楚：格子是点，相邻可走是边，求最少步数是无权最短路。
```

## 8. DFS：深入探索、时间戳与结构发现

DFS 是 depth-first search，深度优先搜索。

核心直觉：

```text
沿着一条路尽量走到底，再回退探索其他选择。
```

### 8.1 DFS 模板

递归版：

```python
def dfs(graph, u, visited):
    visited[u] = True
    for v in graph[u]:
        if not visited[v]:
            dfs(graph, v, visited)
```

完整遍历所有连通分量：

```python
def count_components(graph):
    n = len(graph)
    visited = [False] * n
    count = 0

    for s in range(n):
        if not visited[s]:
            count += 1
            dfs(graph, s, visited)

    return count
```

### 8.2 DFS 的时间戳

DFS 可以记录进入和离开时间：

```python
time = 0

def dfs_time(graph, u):
    global time
    time += 1
    tin[u] = time

    for v in graph[u]:
        if tin[v] == 0:
            parent[v] = u
            dfs_time(graph, v)

    time += 1
    tout[u] = time
```

时间戳可以帮助判断：

```text
祖先后代关系；
边的类型；
有向图是否有环；
拓扑排序；
强连通分量；
桥和割点；
```

### 8.3 DFS 与环检测

有向图中常用三色标记：

```text
0 = 未访问；
1 = 正在递归栈中；
2 = 已完成；
```

```python
def has_cycle_directed(graph):
    n = len(graph)
    color = [0] * n

    def dfs(u):
        color[u] = 1
        for v in graph[u]:
            if color[v] == 1:
                return True
            if color[v] == 0 and dfs(v):
                return True
        color[u] = 2
        return False

    for i in range(n):
        if color[i] == 0 and dfs(i):
            return True
    return False
```

思维重点：

```text
遇到正在递归栈中的点，说明有一条边回到了当前路径上的祖先，于是形成有向环。
```

### 8.4 DFS 和回溯的关系

DFS 用于遍历图。

回溯是 DFS 在状态空间上的应用。

例如排列生成：

```text
状态是当前已选序列；
边是选择下一个元素；
目标是走到完整排列；
```

所以回溯题也可以看成隐式图 DFS。

这就是图论思维的迁移能力。

## 9. 树：最小连通图、无环连通图与递归结构

树是图论中最重要的结构之一。

无向图中的树可以等价定义为：

```text
连通且无环；
任意两个顶点之间有唯一简单路径；
有 n 个顶点且 n - 1 条边，并且连通；
有 n 个顶点且 n - 1 条边，并且无环；
删除任意一条边都会不连通；
加入任意一条新边都会产生环；
```

这些等价定义非常重要。

优秀学生会问：

```text
题目中哪个定义最方便？
是要用“无环”，还是用“唯一路径”，还是用“n-1 条边”？
```

### 9.1 树的边数

任意 n 个顶点的树有 n - 1 条边。

证明思路之一：归纳。

```text
n = 1 时，边数为 0；
任意 n > 1 的树至少有一个叶子；
删除一个叶子和它的边，剩下仍是 n-1 个顶点的树；
由归纳假设，剩下有 n-2 条边；
加回叶子边，总边数 n-1；
```

### 9.2 叶子

树中度为 1 的点叫叶子。

任何至少两个顶点的树至少有两个叶子。

直觉：

```text
从任意点开始沿不回头路径一直走；
因为树无环，不可能无限走；
走到头就是叶子；
从另一端也能找到叶子；
```

### 9.3 根树

选择一个根后，树变成层次结构。

```text
父节点；
子节点；
祖先；
后代；
深度；
子树；
```

很多树算法都是 DFS 的特殊情况。

例如计算子树大小：

```python
def dfs_tree(u, parent):
    size[u] = 1
    for v in tree[u]:
        if v == parent:
            continue
        dfs_tree(v, u)
        size[u] += size[v]
```

### 9.4 生成树

连通图的生成树：

```text
包含原图所有顶点；
使用原图部分边；
形成一棵树；
```

BFS 和 DFS 都能产生生成树。

最小生成树则是在加权连通图中找总权重最小的生成树。

## 10. 二分图：奇环、染色与匹配的入口

二分图是顶点可以分成两组的图：

```text
V = L ∪ R
每条边都连接 L 中一个点和 R 中一个点；
同组内部没有边；
```

### 10.1 二分图的等价刻画

无向图是二分图，当且仅当它没有奇环。

这条定理很重要。

直觉：

```text
如果可以用两种颜色给点染色，使每条边两端颜色不同，那么沿着环走颜色必须交替；
偶数长度环可以回到起点颜色一致；
奇数长度环会要求起点同时是两种颜色，矛盾；
```

### 10.2 BFS 染色判断二分图

```python
from collections import deque

def is_bipartite(graph):
    n = len(graph)
    color = [-1] * n

    for s in range(n):
        if color[s] != -1:
            continue
        color[s] = 0
        q = deque([s])

        while q:
            u = q.popleft()
            for v in graph[u]:
                if color[v] == -1:
                    color[v] = color[u] ^ 1
                    q.append(v)
                elif color[v] == color[u]:
                    return False

    return True
```

### 10.3 二分图建模

典型二分图：

```text
学生 - 项目；
工人 - 任务；
左括号位置 - 右括号位置；
行 - 列；
用户 - 商品；
```

只要问题天然有“两类对象”和“可匹配关系”，就应该想到二分图。

## 11. 有向无环图 DAG 与拓扑排序

DAG 是 directed acyclic graph，有向无环图。

它适合表达依赖关系。

例如：

```text
课程 A 是课程 B 的先修课：A -> B
任务 X 必须在任务 Y 前完成：X -> Y
```

### 11.1 拓扑排序

拓扑排序是 DAG 顶点的线性顺序，使得每条边 `u -> v` 中，`u` 都排在 `v` 前面。

如果有环，就不可能有拓扑排序。

### 11.2 Kahn 算法

核心思想：

```text
入度为 0 的点没有前置依赖，可以先处理；
删除它和它指出的边；
新的入度为 0 的点继续处理；
```

```python
from collections import deque

def topo_sort(graph):
    n = len(graph)
    indeg = [0] * n
    for u in range(n):
        for v in graph[u]:
            indeg[v] += 1

    q = deque([i for i in range(n) if indeg[i] == 0])
    order = []

    while q:
        u = q.popleft()
        order.append(u)
        for v in graph[u]:
            indeg[v] -= 1
            if indeg[v] == 0:
                q.append(v)

    if len(order) != n:
        return None
    return order
```

### 11.3 DFS 拓扑排序

DFS 完成时间逆序也可以得到拓扑序。

直觉：

```text
如果 u -> v，那么 DFS 中 v 会在 u 之前完成，或者 v 已经完成；
所以按完成时间倒序，u 排在 v 前面；
```

### 11.4 DAG 动态规划

DAG 没有环，所以可以按拓扑序做 DP。

例如最长路径：

```python
def longest_path_dag(graph, order):
    n = len(graph)
    dp = [0] * n
    for u in order:
        for v in graph[u]:
            dp[v] = max(dp[v], dp[u] + 1)
    return dp
```

注意：

```text
一般图最长简单路径很难；
DAG 上因为没有环，可以按拓扑顺序动态规划；
```

这体现了结构的重要性。

## 12. 最短路径：BFS、Dijkstra、Bellman-Ford 与 Floyd

最短路径问题根据边权不同选择算法。

| 场景 | 算法 |
| --- | --- |
| 无权图 | BFS |
| 非负边权单源最短路 | Dijkstra |
| 有负权边单源最短路 | Bellman-Ford |
| 所有点对最短路，顶点较少 | Floyd-Warshall |
| DAG 单源最短路 | 拓扑序 DP |

### 12.1 Dijkstra

Dijkstra 适合非负边权。

核心思想：

```text
每次选择当前距离最小、尚未确定的点；
因为边权非负，这个点的最短距离不会再被未来路径改小；
```

优先队列实现：

```python
import heapq

def dijkstra(graph, s):
    n = len(graph)
    INF = 10**30
    dist = [INF] * n
    dist[s] = 0
    pq = [(0, s)]

    while pq:
        d, u = heapq.heappop(pq)
        if d != dist[u]:
            continue
        for v, w in graph[u]:
            if dist[v] > dist[u] + w:
                dist[v] = dist[u] + w
                heapq.heappush(pq, (dist[v], v))

    return dist
```

复杂度：

```text
邻接表 + 二叉堆：O((V + E) log V)
```

### 12.2 Dijkstra 为什么不能有负权边

反例：

```text
s -> a 权重 2
s -> b 权重 5
b -> a 权重 -10
```

Dijkstra 可能先确定 `a` 的距离为 2。

但之后发现 `s -> b -> a` 距离为 -5。

这破坏了“当前最小距离已最终确定”的不变量。

所以 Dijkstra 的关键前提是：

```text
边权非负。
```

### 12.3 Bellman-Ford

Bellman-Ford 可以处理负权边，并能检测负环。

核心思想：

```text
最短路如果没有重复顶点，最多包含 V - 1 条边；
反复松弛所有边 V - 1 轮；
如果第 V 轮仍能松弛，说明存在可达负环；
```

```python
def bellman_ford(n, edges, s):
    INF = 10**30
    dist = [INF] * n
    dist[s] = 0

    for _ in range(n - 1):
        changed = False
        for u, v, w in edges:
            if dist[u] < INF and dist[v] > dist[u] + w:
                dist[v] = dist[u] + w
                changed = True
        if not changed:
            break

    has_neg_cycle = False
    for u, v, w in edges:
        if dist[u] < INF and dist[v] > dist[u] + w:
            has_neg_cycle = True
            break

    return dist, has_neg_cycle
```

复杂度：

```text
O(VE)
```

### 12.4 Floyd-Warshall

Floyd 求所有点对最短路。

状态：

```text
dist[i][j] = 当前允许某些中间点时，i 到 j 的最短距离。
```

转移：

```text
dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])
```

```python
def floyd(dist):
    n = len(dist)
    INF = 10**30
    for k in range(n):
        for i in range(n):
            if dist[i][k] >= INF:
                continue
            for j in range(n):
                if dist[k][j] >= INF:
                    continue
                if dist[i][j] > dist[i][k] + dist[k][j]:
                    dist[i][j] = dist[i][k] + dist[k][j]
    return dist
```

这里假设调用者也用同一个足够大的 `INF` 表示不可达。真实工程中可以把 `INF` 作为参数传入，避免不同模块使用不同哨兵值。

复杂度：

```text
O(V^3)
```

适合顶点数较小的稠密图。

## 13. 最小生成树：割性质、环性质、Kruskal 与 Prim

最小生成树 MST：

```text
在加权连通无向图中，选择 n - 1 条边连接所有顶点，使总权重最小。
```

### 13.1 割性质

割是把顶点分成两部分。

跨越割的边叫 crossing edge。

割性质：

```text
对于任意割，跨越该割的最轻边是某棵最小生成树中的安全边。
```

直觉：

```text
如果一棵 MST 没有选这条最轻边；
它必然选了另一条跨割边来连接两边；
用最轻边替换那条边，总权重不会变大；
所以存在一棵 MST 包含这条最轻边；
```

这就是 Kruskal 和 Prim 的正确性核心。

### 13.2 Kruskal

思想：

```text
按边权从小到大考虑；
如果加入这条边不会形成环，就加入；
直到选够 n - 1 条边；
```

```python
def kruskal(n, edges):
    dsu = DSU(n)
    total = 0
    chosen = []

    for w, u, v in sorted(edges):
        if dsu.union(u, v):
            total += w
            chosen.append((u, v, w))

    if len(chosen) != n - 1:
        return None
    return total, chosen
```

复杂度主要来自排序：

```text
O(E log E)
```

### 13.3 Prim

思想：

```text
从任意起点开始维护已连接集合；
每次选择从集合内部连到外部的最轻边；
```

```python
import heapq

def prim(graph, start=0):
    n = len(graph)
    visited = [False] * n
    pq = [(0, start)]
    total = 0
    count = 0

    while pq:
        w, u = heapq.heappop(pq)
        if visited[u]:
            continue
        visited[u] = True
        total += w
        count += 1

        for v, weight in graph[u]:
            if not visited[v]:
                heapq.heappush(pq, (weight, v))

    if count != n:
        return None
    return total
```

### 13.4 MST 和最短路不要混淆

最小生成树：

```text
目标是以最小总成本连接所有点。
```

最短路：

```text
目标是从源点到某点的路径成本最小。
```

MST 上两点之间的路径不一定是原图最短路。

反例很常见，所以不要看到“最小”就套 MST。

## 14. 并查集：动态连通性与 Kruskal 的底层工具

并查集 DSU 支持：

```text
find(x)：找到 x 所在集合代表；
union(a, b)：合并 a 和 b 所在集合；
connected(a, b)：判断是否同集合；
```

### 14.1 路径压缩和按大小合并

```python
class DSU:
    def __init__(self, n):
        self.parent = list(range(n))
        self.size = [1] * n

    def find(self, x):
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])
        return self.parent[x]

    def union(self, a, b):
        ra = self.find(a)
        rb = self.find(b)
        if ra == rb:
            return False
        if self.size[ra] < self.size[rb]:
            ra, rb = rb, ra
        self.parent[rb] = ra
        self.size[ra] += self.size[rb]
        return True
```

均摊复杂度接近常数。

### 14.2 并查集适合的问题

```text
无向图动态加边连通性；
Kruskal 最小生成树；
判断加入边是否成环；
岛屿数量动态变化；
等价关系合并；
```

### 14.3 并查集不适合的问题

普通并查集不擅长：

```text
删除边；
有向可达性；
最短路径；
输出具体路径；
```

这是数据结构边界。

优秀学生会问：

```text
我的操作是只加边，还是有删边？
我要的是连通性，还是路径长度？
图是无向还是有向？
```

## 15. 强连通分量：有向图中的互相可达结构

有向图中，强连通分量 SCC 是最大的一组互相可达顶点。

### 15.1 为什么 SCC 重要

SCC 可以把有向图压缩成 DAG。

```text
每个 SCC 缩成一个点；
不同 SCC 之间如果有边，则连边；
得到的缩点图一定是 DAG；
```

这非常有用：

```text
分析依赖循环；
判断软件模块循环依赖；
网页链接结构；
2-SAT；
有向图 DP；
```

### 15.2 Kosaraju 思想

Kosaraju 两遍 DFS：

```text
第一遍在原图上 DFS，记录完成时间；
第二遍在反图上按完成时间逆序 DFS；
每次 DFS 得到一个 SCC；
```

直觉：

```text
完成时间帮助我们先从凝聚 DAG 的源或汇方向切入；
反图把可达方向反过来；
第二遍 DFS 刚好收集一个强连通块；
```

### 15.3 Tarjan 思想

Tarjan 用一次 DFS 和 lowlink。

核心概念：

```text
dfn[u]：u 被 DFS 访问的时间；
low[u]：u 能通过 DFS 树边和返祖边到达的最早 dfn；
栈：维护当前尚未归属 SCC 的点；
```

当：

```text
low[u] == dfn[u]
```

说明 u 是一个 SCC 的根，可以从栈中弹出该 SCC。

初学者不必一开始死背 Tarjan。

先理解：

```text
SCC 是有向图互相可达的块；
缩点后是 DAG；
```

再学具体实现会更稳。

## 16. 欧拉图：一次走完所有边

欧拉问题关心边。

经典问题：

```text
能否从某点出发，每条边恰好走一次？
```

### 16.1 欧拉回路和欧拉路径

欧拉回路：

```text
每条边恰好经过一次，最后回到起点。
```

欧拉路径：

```text
每条边恰好经过一次，但不要求回到起点。
```

### 16.2 无向图判定

忽略孤立点后图连通。

欧拉回路条件：

```text
所有顶点度数都是偶数。
```

欧拉路径条件：

```text
恰好 0 个或 2 个奇数度顶点。
```

为什么？

```text
每次进入一个中间点，就必须从它离开；
所以中间点使用的边成对出现，度数为偶数；
如果不是回路，起点和终点可以各多出一条未配对边；
因此奇度点只能是 0 或 2 个；
```

### 16.3 Hierholzer 算法

思想：

```text
沿着未使用边走，形成回路或路径；
如果当前路径上还有点连接未使用边，从那里继续插入新回路；
```

复杂度：

```text
O(V + E)
```

### 16.4 欧拉和哈密顿的区别

欧拉：

```text
每条边一次。
```

哈密顿：

```text
每个顶点一次。
```

欧拉问题有漂亮的度数判定。

哈密顿问题通常困难得多。

## 17. 哈密顿问题：一次访问所有点为什么更难

哈密顿路径：

```text
经过每个顶点恰好一次的路径。
```

哈密顿回路：

```text
经过每个顶点恰好一次并回到起点的回路。
```

### 17.1 为什么不能像欧拉那样简单判定

欧拉问题看边，每个点的进出可以用度数局部控制。

哈密顿问题看点，限制更全局：

```text
一个局部度数条件不一定决定是否存在访问所有点的路径；
```

有很多充分条件，例如 Dirac 定理、Ore 定理，但它们不是必要条件。

初学阶段要记住：

```text
哈密顿问题一般比欧拉问题难很多。
```

### 17.2 小规模哈密顿 DP

旅行商问题 TSP 的状态压缩 DP：

```text
dp[mask][v] = 从起点出发，访问 mask 中点，最后到 v 的最小代价。
```

复杂度：

```text
O(2^n n^2)
```

这说明：

```text
当 n 很小时，可以用指数 DP；
当 n 很大时，需要利用特殊结构、近似、剪枝或改变问题。
```

## 18. 平面图：画在平面上不交叉的图

平面图是可以画在平面上，使边只在端点相交的图。

注意：

```text
平面图不是指你随手画出来没有交叉；
而是存在某种画法没有交叉。
```

### 18.1 欧拉公式

连通平面图满足：

```text
V - E + F = 2
```

其中：

```text
V 是顶点数；
E 是边数；
F 是面数，包括外部无限面；
```

这是图论中非常重要的结构公式。

### 18.2 平面简单图边数上界

若简单连通平面图 `V >= 3`，则：

```text
E <= 3V - 6
```

如果没有三角形，则：

```text
E <= 2V - 4
```

这些结论常用于证明某些图非平面。

### 18.3 K5 和 K3,3

两个经典非平面图：

```text
K5：5 个点完全图；
K3,3：三个房子三个水电气问题中的完全二分图；
```

更深的 Kuratowski 定理说明，非平面性和包含这两类图的细分有关。

初学阶段重点是：

```text
会用欧拉公式和边数上界构造非平面证明。
```

## 19. 图着色：冲突建模与资源分配

图着色问题：

```text
给顶点染色，使相邻顶点颜色不同。
```

最少需要的颜色数叫色数。

### 19.1 着色建模

典型场景：

| 问题 | 顶点 | 边 | 颜色 |
| --- | --- | --- | --- |
| 考试排期 | 考试 | 同一学生参加两门考试 | 时间段 |
| 寄存器分配 | 变量 | 生命周期重叠 | 寄存器 |
| 地图着色 | 区域 | 共享边界 | 颜色 |
| 任务冲突 | 任务 | 不能同时做 | 批次 |

图着色本质是冲突分组。

### 19.2 二分图是 2-着色图

一个图可以用 2 种颜色染色，当且仅当它是二分图。

所以二分图判断也是图着色的基础特例。

### 19.3 贪心着色

按某个顺序给每个点选最小可用颜色：

```python
def greedy_coloring(graph, order):
    n = len(graph)
    color = [-1] * n
    for u in order:
        used = set()
        for v in graph[u]:
            if color[v] != -1:
                used.add(color[v])
        c = 0
        while c in used:
            c += 1
        color[u] = c
    return color
```

注意：

```text
贪心着色结果依赖顺序；
不保证最优；
但在工程中常作为启发式；
```

### 19.4 四色定理

任何平面地图都可以用 4 种颜色染色，使相邻区域颜色不同。

这是著名定理，但证明很复杂。

初学者更应该掌握：

```text
如何把冲突建模成着色；
如何判断二分图；
如何用贪心算法得到可行染色；
为什么最优着色通常不简单；
```

## 20. 匹配：从二分图配对到最大匹配

匹配是边的集合，其中任意两条边不共享端点。

最大匹配：

```text
边数最多的匹配。
```

完美匹配：

```text
每个顶点都被匹配覆盖。
```

### 20.1 二分图匹配建模

例子：

```text
左边是学生；
右边是项目；
如果学生愿意做某项目，就连边；
目标是最多学生被分配到不同项目；
```

这就是二分图最大匹配。

### 20.2 增广路思想

给定一个匹配。

增广路是：

```text
从未匹配左点开始；
边在“非匹配边、匹配边、非匹配边...”之间交替；
到达未匹配右点；
```

沿增广路翻转边：

```text
原来不在匹配中的边加入；
原来在匹配中的边移除；
匹配大小增加 1；
```

核心定理：

```text
一个匹配是最大匹配，当且仅当不存在增广路。
```

这是一种非常 MIT/CMU 风格的思想：

```text
如果当前解不是最优，就能找到局部结构改进它；
如果找不到改进结构，就证明它已经最优。
```

### 20.3 DFS 匈牙利算法简化版

```python
def max_bipartite_matching(adj, n_left, n_right):
    match_r = [-1] * n_right

    def dfs(u, seen):
        for v in adj[u]:
            if seen[v]:
                continue
            seen[v] = True
            if match_r[v] == -1 or dfs(match_r[v], seen):
                match_r[v] = u
                return True
        return False

    ans = 0
    for u in range(n_left):
        seen = [False] * n_right
        if dfs(u, seen):
            ans += 1
    return ans, match_r
```

复杂度简化估计：

```text
O(VE)
```

更高效的二分图最大匹配可用 Hopcroft-Karp。

## 21. 网络流：容量、割与最大流最小割

网络流模型：

```text
有向图；
每条边有容量；
有源点 s 和汇点 t；
目标是从 s 向 t 发送尽可能多的流；
每条边流量不能超过容量；
中间点流入等于流出；
```

### 21.1 流网络的直觉

可以把它想成管道网络：

```text
边是管道；
容量是管道最大输送量；
s 是水源；
t 是出口；
中间节点不能凭空产生或消灭流量；
```

### 21.2 残量网络

如果一条边 `u -> v` 容量为 10，已经用了 6。

那么：

```text
正向残量 = 4；
反向残量 = 6；
```

反向边表示：

```text
允许撤销之前的一部分流量安排。
```

这是网络流最容易被初学者忽略的关键。

### 21.3 Ford-Fulkerson 增广思想

```text
只要残量网络中存在 s 到 t 的路径；
就沿这条路径增加尽可能多的流；
直到不存在增广路径；
```

Edmonds-Karp 用 BFS 找最短增广路径，复杂度：

```text
O(VE^2)
```

### 21.4 最大流最小割

割把顶点分成 S 和 T：

```text
s 在 S；
t 在 T；
```

割容量是从 S 指向 T 的边容量总和。

最大流最小割定理：

```text
最大 s-t 流量 = 最小 s-t 割容量。
```

直觉：

```text
任何流都不能超过任意割的容量；
如果找不到增广路，残量网络中 s 可达集合形成一个割；
这个割容量刚好等于当前流量；
所以上界被达到，当前流最大；
```

这是算法正确性证明中的经典“上界 + 构造达到上界”。

## 22. 图论证明方法：归纳、极值、反证、构造与不变量

图论不是只有算法，还有大量证明。

### 22.1 归纳法

适合树、平面图、递归结构。

例：证明树有 n - 1 条边。

方法：

```text
找到叶子；
删除叶子；
对更小的树使用归纳假设；
加回叶子；
```

### 22.2 极值法

选择“最长路径”“最小反例”“最大匹配”等极端对象。

例如证明树至少有两个叶子：

```text
取树中一条最长简单路径；
如果端点不是叶子，还能继续延长；
矛盾；
所以两个端点都是叶子；
```

### 22.3 反证法

假设结论不成立，然后推出矛盾。

例如：

```text
如果奇数度顶点个数为奇数；
那么所有度数之和为奇数；
但握手定理说明它等于 2E，是偶数；
矛盾；
```

### 22.4 构造法

直接构造对象。

例如：

```text
要证明某图有生成树；
可以从任意顶点开始做 DFS 或 BFS；
遍历过程中记录第一次发现每个点的边；
这些边构成生成树；
```

### 22.5 不变量

算法证明经常靠不变量。

例 BFS：

```text
队列按距离非递减；
第一次发现的距离就是最短距离；
```

例 Kruskal：

```text
当前选择的边集始终可以扩展成某棵 MST；
每次加入的边都是安全边；
```

例 Dijkstra：

```text
已经弹出并确认的点，其 dist 就是最终最短距离；
```

学习图算法时，不变量比代码模板更重要。

## 23. 图论建模：如何把陌生问题翻译成图

建模是图论最有价值的能力。

### 23.1 建模清单

拿到题目后问：

```text
顶点是什么？
边是什么？
边是有向还是无向？
边是否有权重？
权重表示距离、成本、时间、容量还是概率？
是否需要多源点或超级源点？
是否有两类对象，可以建二分图？
是否是依赖关系，可以建 DAG？
是否是冲突关系，可以建补图或着色图？
是否是“最多配对”或“最大运输”，可以建匹配或流？
```

### 23.2 状态空间图

很多题没有显式给图，但可以把状态当顶点。

例如：

```text
密码锁从 0000 变到 1234，每次转一位；
```

建模：

```text
顶点：每个四位状态；
边：一次合法旋转；
目标：最少操作次数；
算法：BFS；
```

### 23.3 依赖图

如果题目出现：

```text
必须先；
依赖；
前置条件；
拓扑顺序；
能否完成；
```

通常建有向图。

如果要判断能否完成：

```text
检测是否有环。
```

如果要输出完成顺序：

```text
拓扑排序。
```

### 23.4 冲突图

如果题目出现：

```text
不能同时；
互斥；
相邻不能同色；
同一资源不能共享；
```

通常建冲突图。

目标可能是：

```text
二分图判断；
图着色；
最大独立集；
最小颜色数；
```

注意有些冲突图问题很难，不能默认有简单多项式算法。

### 23.5 补图思维

有时边表示“不能一起”。

有时更方便让边表示“可以一起”。

这时可以考虑补图。

例如：

```text
找一组两两不冲突的人；
在冲突图中是独立集；
在非冲突补图中是团；
```

图论建模常常取决于你选择让边表达什么。

## 24. 常见误区与反例意识

### 24.1 把 BFS 用在加权图最短路

BFS 最短路只适合无权图或等权图。

只要边权不同，就要重新选择算法。

### 24.2 以为 DFS 第一次到达就是最短路

DFS 第一次到达某点不保证最短。

DFS 适合探索结构，不适合无权最短路，除非你遍历所有路径并做额外比较。

### 24.3 把 MST 当成最短路树

MST 总成本最小，不保证任意两点之间距离最短。

最短路树从某个源点出发保证源点到其他点距离最短，但总权重不一定最小生成。

### 24.4 忽略图是否连通

很多定理默认连通。

例如：

```text
树是连通无环图；
欧拉路径判定要求忽略孤立点后连通；
MST 要求图连通，否则只有最小生成森林；
```

### 24.5 忽略方向

有向图和无向图差别很大。

例如：

```text
无向连通不等于有向强连通；
拓扑排序只适用于有向图；
SCC 只在有向图中有意义；
```

### 24.6 忽略重边和自环

有些算法对重边没问题，有些定理默认简单图。

例如平面简单图边数上界 `E <= 3V - 6` 默认简单图。

题目前提不清时，要主动确认。

### 24.7 认为局部最优一定全局最优

图算法里贪心能成功，通常需要结构性质。

例如：

```text
MST 有割性质；
Dijkstra 有非负边权；
```

没有证明结构时，贪心可能失败。

## 25. 综合例题拆解

### 25.1 例题一：课程能否完成

题目：

```text
有 n 门课程和若干先修关系 [a, b]，表示学 a 前必须学 b。
问能否完成所有课程。
```

建模：

```text
顶点：课程；
边：b -> a；
目标：判断有向图是否有环；
算法：拓扑排序或 DFS 三色；
```

Kahn 算法思路：

```text
入度为 0 的课可以先学；
学完后减少后继课程入度；
如果最终学完 n 门，说明无环；
否则有环；
```

复杂度：

```text
O(V + E)
```

### 25.2 例题二：岛屿数量

题目：

```text
网格中 1 表示陆地，0 表示水，四方向相邻陆地属于同一岛屿。
求岛屿数量。
```

建模：

```text
每个陆地格子是顶点；
相邻陆地之间有边；
目标：连通分量数量；
算法：DFS/BFS；
```

这不是“网格题模板”，而是“隐式图连通分量”。

### 25.3 例题三：连接所有城市最低成本

题目：

```text
n 个城市，若干可修道路及成本，求连接所有城市的最低总成本。
```

建模：

```text
顶点：城市；
边：可修道路；
权重：成本；
目标：最小成本连接所有顶点；
算法：最小生成树；
```

如果图不连通：

```text
无解，或只能得到最小生成森林；
```

### 25.4 例题四：最少基因变换次数

题目：

```text
每次只能改变一个字符，且中间字符串必须在基因库里。
求从 start 到 end 最少变换次数。
```

建模：

```text
顶点：基因字符串；
边：两个字符串只差一个字符；
目标：最少边数；
算法：BFS；
```

优化思考：

```text
直接两两比较可能 O(N^2 L)；
可以用通配模式或哈希加速邻居生成；
```

### 25.5 例题五：工人与任务分配

题目：

```text
若干工人和任务，每个工人能做部分任务，每个任务最多分给一人。
求最多能完成多少任务。
```

建模：

```text
左侧顶点：工人；
右侧顶点：任务；
边：工人能做任务；
目标：最大匹配；
算法：二分图匹配或最大流；
```

如果每个任务有容量、每个工人能做多个任务：

```text
模型可能升级为网络流。
```

## 26. 训练路线与练习计划

### 26.1 第一阶段：定义和小证明

练习：

```text
证明握手定理；
证明奇数度顶点个数为偶数；
证明树有 n - 1 条边；
证明树中任意两点有唯一简单路径；
证明二分图无奇环；
```

目标：

```text
能用精确定义写出短证明。
```

### 26.2 第二阶段：遍历和连通

练习：


```text

每个算法都写出不变量和正确性证明；
每个定理都尝试找反例，确认条件不可随便删；

```

真正掌握图论的标志，不是记住某个模板，而是能解释：

```text

为什么 BFS 第一次到达是无权最短路；
为什么 Dijkstra 需要非负边权；
为什么 MST 的贪心选择是安全的；
为什么二分图等价于没有奇环；
为什么 SCC 缩点后一定是 DAG；
为什么网络流的最大值会等于最小割；

```
