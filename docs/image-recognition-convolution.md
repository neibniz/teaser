## 1. 图像识别到底在做什么

图像识别的任务是：

> 给定一张图片，让模型判断图片中有什么。

常见任务：

| 任务 | 输出 |
| --- | --- |
| 图像分类 | 这是一只猫 |
| 目标检测 | 猫在哪里，狗在哪里 |
| 图像分割 | 每个像素属于什么物体 |
| 人脸识别 | 这是谁 |
| OCR | 图片里的文字是什么 |

最基础的是图像分类：

```text
输入：图片 x
输出：类别 y
模型：f(x) = y
```

模型通常输出概率：

```text
P(猫 | 图片) = 0.92
P(狗 | 图片) = 0.04
P(车 | 图片) = 0.01
```

最后选概率最大的类别。

### 1.1 一流课堂会先问什么

优秀教授通常不会先问“用哪个网络”，而会问：

- 图片的输入大小是多少？
- 是灰度图还是彩色图？
- 类别是否互斥？
- 背景是否会误导模型？
- 训练数据和真实场景是否一致？
- 错误分类的代价是否一样？
- 模型要识别物体、纹理、颜色，还是局部缺陷？

优秀学生会先理解任务，再选择模型。

## 2. 图像在计算机中是什么

对人来说，图片是物体、颜色和场景。  
对计算机来说，图片是数字矩阵。

### 2.1 灰度图

灰度图只有一个通道。

一张 `4 x 4` 灰度图可以表示为：

```text
[
  [  0,  20,  40,  60],
  [ 10,  30,  50,  70],
  [100, 120, 140, 160],
  [110, 130, 150, 170]
]
```

每个数字表示一个像素的亮度。

通常：

```text
0   -> 黑
255 -> 白
```

### 2.2 彩色图

彩色图片一般有 3 个通道：

```text
R 红色
G 绿色
B 蓝色
```

一张图片可以看成：

```text
height x width x channels
```

例如：

```text
224 x 224 x 3
```

表示高度 224、宽度 224、3 个颜色通道。

### 2.3 图像识别的难点

同一只猫可能：

- 出现在不同位置。
- 有不同大小。
- 有不同姿态。
- 光照不同。
- 背景不同。
- 被部分遮挡。

模型必须学会：

> 不被表面像素变化迷惑，而抓住稳定的视觉模式。

## 3. 从滤波器理解卷积

卷积可以先理解成：

> 用一个小窗口在图片上滑动，检查每个局部区域是否像某种模式。

这个小窗口叫卷积核，也叫滤波器。

### 3.1 一个边缘检测例子

假设有一个简单卷积核：

```text
[-1, 1]
```

它会比较相邻像素差异。

如果左边暗、右边亮，结果大。  
如果左右差不多，结果小。

所以它能检测水平位置上的亮度变化，也就是边缘。

### 3.2 卷积核像一个模式探测器

可以把卷积核想成一个小模板。

它在图片上每个位置问：

> 这里像不像我想找的模式？

如果像，输出值大。  
如果不像，输出值小。

不同卷积核可以学到不同模式：

- 横向边缘。
- 纵向边缘。
- 斜线。
- 角点。
- 纹理。
- 颜色组合。

初学者要记住：

> 卷积不是神秘公式，而是局部模式匹配。

## 4. 卷积的数学形式

假设输入是一小块 `3 x 3` 图像：

```text
X =
[
  [1, 2, 3],
  [4, 5, 6],
  [7, 8, 9]
]
```

卷积核：

```text
K =
[
  [1, 0],
  [0, -1]
]
```

窗口覆盖左上角 `2 x 2` 区域：

```text
[
  [1, 2],
  [4, 5]
]
```

对应位置相乘再相加：

```text
1*1 + 2*0 + 4*0 + 5*(-1) = -4
```

这就是一个输出值。

窗口继续滑动，就得到一张新的矩阵，叫特征图。

### 4.1 公式

二维卷积可以写成：

```text
Y[i, j] = sum_m sum_n X[i+m, j+n] * K[m, n]
```

含义：

- `X` 是输入图像。
- `K` 是卷积核。
- `Y` 是输出特征图。
- `i, j` 表示卷积核滑到的位置。

### 4.2 一流课堂会强调什么

教授会让你解释：

> 公式里的每一项在图像中代表什么？

如果你只能背公式，不知道窗口在滑动，就没有真正理解。

优秀学生会把公式翻译成语言：

> 在每个位置，把局部像素和卷积核逐项相乘并求和，得到这个位置对某种模式的响应强度。

## 5. 为什么卷积适合图像

卷积适合图像，主要因为图像有三个重要性质。

### 5.1 局部性

图像中的局部像素关系很重要。

例如边缘、角点、纹理，通常由相邻像素决定。

卷积核只看一个小区域，正好符合图像局部结构。

### 5.2 参数共享

同一个卷积核在整张图片上滑动。

这意味着：

> 如果一个边缘检测器在左上角有用，在右下角也应该有用。

不用为每个位置单独学习一套参数。

这大大减少参数量。

### 5.3 平移等变性

如果图片中的物体移动了一点，卷积输出中的响应也会移动。

这叫平移等变性：

```text
输入平移 -> 特征图也相应平移
```

这对图像识别很重要，因为物体不一定固定出现在同一个位置。

### 5.4 与全连接层对比

如果把 `224 x 224 x 3` 图片直接接全连接层，输入维度是：

```text
224 * 224 * 3 = 150528
```

参数会非常多，并且失去空间结构。

卷积则保留空间关系，并通过参数共享降低复杂度。

## 6. 通道、特征图与多个卷积核

### 6.1 输入通道

彩色图片有 3 个通道：

```text
R, G, B
```

卷积核也要覆盖所有输入通道。

如果输入是：

```text
H x W x 3
```

一个卷积核可能是：

```text
3 x 3 x 3
```

它同时看空间位置和颜色通道。

### 6.2 输出通道

一个卷积核产生一张特征图。  
多个卷积核产生多张特征图。

如果有 64 个卷积核，输出就是：

```text
H' x W' x 64
```

每个输出通道可以理解为一种模式探测结果。

### 6.3 特征图是什么

特征图不是原图，而是：

> 某种视觉模式在每个位置的响应强度图。

例如：

- 第 1 张特征图可能响应横向边缘。
- 第 2 张特征图可能响应绿色纹理。
- 第 20 张特征图可能响应圆形结构。

深层特征图会更抽象。

## 7. 步幅、填充与感受野

### 7.1 步幅

步幅表示卷积核每次移动多少格。

```text
stride = 1：每次移动 1 格
stride = 2：每次移动 2 格
```

步幅越大，输出尺寸越小。

### 7.2 填充

如果不填充，卷积后图像会变小。

为了保持尺寸，常在边缘补 0：

```text
padding = 1
```

对于 `3 x 3` 卷积，padding 1 常能保持高度宽度不变。

### 7.3 感受野

感受野是：

> 输出特征中的一个位置，能看到原图中多大的区域。

第一层 `3 x 3` 卷积只能看局部 `3 x 3`。  
多层堆叠后，深层神经元能看到更大区域。

这解释了 CNN 的层级理解：

- 浅层看边缘。
- 中层看局部部件。
- 深层看整体结构。

## 8. 激活函数与池化

### 8.1 为什么需要激活函数

卷积本身是线性运算。

如果所有层都只是线性运算，多层叠加仍然等价于一层线性变换。

所以需要非线性激活函数。

最常见的是 ReLU：

```text
ReLU(x) = max(0, x)
```

含义：

- 正响应保留。
- 负响应压成 0。

直觉：

> 某个模式如果没有出现，就不要给它负激活，让它安静。

### 8.2 池化

最大池化：

```text
在一个小区域中取最大值
```

例如：

```text
[
  [1, 3],
  [2, 4]
]
```

最大池化结果是：

```text
4
```

### 8.3 池化的作用

池化可以：

- 降低尺寸。
- 减少计算。
- 提供一定平移鲁棒性。
- 保留最强特征响应。

但现代 CNN 不一定依赖池化，也可以用步幅卷积降低尺寸。

## 9. CNN 如何逐层识别图像

CNN 的识别过程可以理解为层级组合。

### 9.1 第一层：简单边缘

浅层卷积常学到：

- 横线。
- 竖线。
- 斜线。
- 颜色对比。

这些是图像最基础的结构。

### 9.2 中间层：局部形状

中间层可能组合边缘形成：

- 角。
- 圆弧。
- 纹理。
- 眼睛局部。
- 车轮局部。

### 9.3 深层：高级语义

深层可能响应：

- 猫脸。
- 狗耳朵。
- 车头。
- 飞机机翼。

教授式总结：

> CNN 不是一眼看到“猫”，而是逐层把像素组合成边缘、部件，再组合成对象。

### 9.4 优秀学生会问的问题

- 第一层为什么经常像边缘检测器？
- 多层卷积如何扩大感受野？
- 为什么深层特征更抽象？
- 如果数据很少，深层模型会不会记住训练集？
- 如果背景和类别强相关，模型学到的是物体还是背景？

## 10. 训练 CNN：模型到底在学什么

CNN 的卷积核不是人工手写的，而是通过数据学习出来的。

训练流程：

1. 输入图片。
2. 前向传播得到预测。
3. 计算损失。
4. 反向传播计算梯度。
5. 更新卷积核和其他参数。

### 10.1 损失函数

分类任务常用交叉熵：

```text
Loss = -log(模型给真实类别的概率)
```

如果真实类别是猫：

```text
P(猫) = 0.95 -> 损失小
P(猫) = 0.05 -> 损失大
```

### 10.2 梯度下降

模型参数更新：

```text
参数 = 参数 - 学习率 * 梯度
```

梯度告诉模型：

> 哪些参数应该往哪个方向调整，才能让损失下降。

### 10.3 卷积核如何被学出来

一开始卷积核是随机的。  
如果某个卷积核的响应有助于正确分类，训练会强化它。  
如果某个响应导致错误，训练会调整它。

经过大量样本后，卷积核逐渐变成有效的视觉模式检测器。

### 10.4 数据增强

为了让模型不要记住固定样子，可以做数据增强：

- 随机裁剪。
- 水平翻转。
- 颜色扰动。
- 旋转。
- 缩放。
- Cutout、Mixup 等。

数据增强的直觉：

> 告诉模型：这些变化不应该改变类别。

这相当于把人类先验注入训练过程。

## 11. 经典 CNN 架构的思想演进

### 11.1 LeNet

早期 CNN，用于手写数字识别。  
证明了卷积、池化、全连接组合可以处理图像。

### 11.2 AlexNet

让深度学习图像识别大规模成功。  
关键包括：

- 更深网络。
- ReLU。
- GPU 训练。
- 数据增强。
- Dropout。

### 11.3 VGG

使用大量 `3 x 3` 小卷积堆叠。

思想：

> 用小卷积多层组合，既保持表达能力，又控制参数。

### 11.4 ResNet

ResNet 引入残差连接：

```text
输出 = F(x) + x
```

它解决了深层网络难训练的问题。

直觉：

> 如果某些层暂时学不好，至少可以先接近恒等映射，不要破坏原信息。

### 11.5 一流课堂的总结

架构演进不是为了堆名词，而是在回答问题：

- 如何更深？
- 如何更稳？
- 如何更快？
- 如何减少参数？
- 如何提高泛化？

## 12. 图像识别的常见问题与工程判断

### 12.1 过拟合

训练集准确率高，测试集准确率低，说明可能过拟合。

原因：

- 数据太少。
- 模型太大。
- 标签噪声。
- 训练集和测试集分布不同。

应对：

- 数据增强。
- 正则化。
- Dropout。
- 早停。
- 使用预训练模型。

### 12.2 背景偏差

模型可能学到背景，而不是物体。

例如训练集中牛总是在草地上，模型可能把草地当作牛的证据。

优秀学生会问：

> 模型预测正确，是因为看到了真正的物体，还是因为看到了数据里的偶然相关？

### 12.3 类别不平衡

如果某类图片很少，模型可能忽略它。

处理方式：

- 重采样。
- 类别权重。
- 收集更多少数类数据。
- 使用更合适的指标。

### 12.4 迁移学习

如果数据不多，常用预训练模型。

流程：

1. 使用在大数据集上训练过的 CNN。
2. 替换最后的分类层。
3. 在自己的数据上微调。

直觉：

> 边缘、纹理、形状等底层视觉能力可以迁移到新任务。

## 13. 教授式学习路线

### 第一阶段：像素和矩阵

你应该能回答：

- 图片为什么是矩阵？
- 灰度图和彩色图有什么区别？
- 通道是什么意思？

### 第二阶段：手算卷积

你应该能手算一个 `3 x 3` 输入和 `2 x 2` 卷积核的输出。

目标：

> 把卷积从抽象公式变成具体滑动窗口。

### 第三阶段：理解 CNN 组件

你应该能解释：

- 卷积。
- ReLU。
- 池化。
- 全连接。
- Softmax。
- 损失函数。

### 第四阶段：理解层级特征

你应该能说清：

```text
像素 -> 边缘 -> 纹理/局部形状 -> 部件 -> 物体
```

### 第五阶段：理解训练和泛化

你应该能分析：

- 为什么数据增强有用？
- 为什么预训练有用？
- 为什么训练准确率高不代表模型好？
- 为什么模型可能学到背景偏差？

## 一句话总结

卷积神经网络的核心思想是：

> 用可学习的小窗口在整张图上寻找局部模式，再通过多层组合把像素变成边缘、部件和物体。

从数学上看，卷积是局部加权求和。  
从认知上看，卷积是模式探测。  
从工程上看，卷积利用了图像的局部性、参数共享和平移结构。
